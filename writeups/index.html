Writeup — Offensive Security Intro

Page récapitulative — étapes, observations et recommandations pour la room "Offensive Security Intro" (TryHackMe).

Nom de la room : Offensive Security Intro Lien de la room : https://tryhackme.com/room/offensivesecurityintro Difficulté : easy Durée estimée : ~30 min – 1h Chose(s) faite(s) dans la room : - Lancement de la machine et lecture de l’énoncé. - Reconnaissance réseau et web (scan des ports, découverte de pages cachées). - Exploration d’un site web “FakeBank”, test de paramètres et actions (ex. “Send Money”) pour révéler indices. - Utilisation d’outils basiques (gobuster pour directories, curl/navigateur pour tests). - Vérifications post-compromise simples (si accès fourni) et compréhension de la méthodologie offensive. Étapes dans cette room : 1. Démarrage de la VM et prise en main de l’énoncé (objectifs). 2. Recon : scan rapide des ports/services avec `nmap` pour connaitre la surface d’attaque. 3. Enumeration web : lancer `gobuster`/`ffuf` pour trouver pages/répertoires cachés (ex : /admin, /transfer, etc.). 4. Inspection manuelle des pages trouvées : tester formulaires, paramètres et comportements (ex : effectuer une opération “Send Money” pour déclencher un indice). 5. Si des credentials ou vecteurs d’accès apparaissent, tenter l’accès (SSH ou autre) et examiner le système. (La room montre surtout la logique d’attaque web pour débutants.) 6. Post-exploitation basique : checks initiaux (`sudo -l`, fichiers sensibles, versions) et noter les corrections à recommander. 7. Terminer la machine et noter les leçons apprises. 8. Nettoyage / notes : éléments à garder en tête pour la remediation. Observations & résultats rapides : - La room se concentre sur la démarche (recon → énumération web → test d’hypothèse) plutôt que sur un exploit complexe ; l’important est d’apprendre la logique et les outils de base. - Outils utiles observés : `nmap`, `gobuster` (ou `ffuf`), `curl`/navigateur et parfois `sqlmap` si une injection est suspectée. - Difficulté : conçue pour les débutants — rapide et pédagogique (bon pour se familiariser avec le flow d’un pentest). Remarque(s) / correction(s) recommandée(s) : - Sur une vraie appli : valider/filtrer strictement les entrées côté serveur (requêtes préparées, validation stricte). - Masquer les informations sensibles dans les messages de retour et restreindre l’accès aux pages d’administration. - Logger sans divulguer d’informations exploitables et limiter les fonctionnalités dangereuses exposées. - Mettre à jour et patcher les services vulnérables ; restreindre SUID et privilèges inutiles. Outils utilisés : gobuster, Kali (prototype) — et outils standards : nmap, curl, sqlmap (si besoin). Conclusion : Excellente room d’introduction pour comprendre la méthodologie offensive — elle permet de pratiquer la reconnaissance web, la recherche de répertoires et la validation d’hypothèses, base du pentest. 🚀
